# LINE秘書ブリーフィング 設計書

作成日: 2026-06-26
対象リポジトリ: `/home/yuta/line/line-news-bot`

## 1. 概要

既存の「パーソナライズニュース自動配信システム」(`news_delivery.py`) を退役させ、
朝5:30に **1通だけ** 届く「フランクな秘書からの朝の通達」へ作り変える。

メッセージは4ブロック構成：
1. 📅 予定リマインド（Googleカレンダー）
2. ☀️ 天気（自宅・座間／職場・港区）
3. 📰 今朝の一報（今日のニュース1件）
4. 📖 本日の一語（教養になる単語1つ）

口調は **フランクな秘書**（タメ口寄りだが世話焼きで丁寧さも残す）。

## 2. メッセージ例

```
おはよう、渡辺さん。6月26日（金）の朝の通達だよ。

📅 今日の予定
・14:00 歯科検診（▶本日）
・6/29 友人と食事（▶3日後）

☀️ 天気
〔自宅・座間〕晴れ 28/19℃ ☔10%
〔職場・港区〕くもり 27/20℃ ☔40%
傘はたぶん要らないよ。

📰 今朝の一報
・〇〇法案が衆院通過（NHK）

📖 本日の一語
「白眉（はくび）」
意味：多くの中で最も優れた人・もの。
由来：蜀の馬氏五兄弟のうち眉に白毛のある長兄が最も優秀だった故事から。

今日もご安全に。
```

予定ブロックの表示ルール：
- **当日分**：今日の予定があれば一覧表示。無ければ「本日の予定はございません」と表示。
- **事前通達（7日後 / 3日後 / 1日後）**：該当する予定が**存在する場合のみ**行を追加する。
  該当ゼロなら何も足さない（「予定なし」の予告はしない）。

## 3. 仕様詳細

### 3.1 予定リマインド (`calendar_events.py`)
- 取得方法：Googleカレンダーの **iCal秘密URL**（`.ics`）。`.env` に `CALENDAR_ICAL_URL` として保存。
- 認証不要・読み取り専用。OAuth/トークン更新は使わない。
- 抽出ロジック：今日を基準に、開始日が **7日後 / 3日後 / 1日後 / 当日** のいずれかに該当する予定だけを拾う。
  - 各予定に該当ラベルを付ける：`▶本日` / `▶明日` / `▶3日後` / `▶1週間後`。
- **表示ルール**：
  - 当日に該当する予定がなければ「本日の予定はございません」を出す。
  - 事前通達（7/3/1日後）は **該当する予定があるときだけ** 行を追加。該当ゼロなら何も出さない（不在の予告はしない）。
- 繰り返し予定（RRULE）にも対応する（`recurring-ical-events` を利用）。
- 終日予定と時刻付き予定の両方を扱う。時刻付きは `HH:MM` を表示、終日は時刻なし。
- タイムゾーンは JST 基準で判定。

### 3.2 天気 (`weather.py`)
- API：**Open-Meteo**（無料・APIキー不要）。
- 2地点：自宅＝神奈川県座間市、職場＝東京都港区。
- 座標は設定に固定値で埋め込む（ジオコーディングAPI往復を避け、安定化）。
  - 座間市：緯度 35.4886, 経度 139.4078
  - 港区：緯度 35.6581, 経度 139.7516
- 取得項目：今日の天気概況・最高/最低気温・降水確率。
  - （余裕があれば明日も。MVPは今日のみで可）
- 天気コード(WMO)→日本語＋絵文字に変換するテーブルを持つ。
- 降水確率に応じて一言コメント（「傘いるよ」/「傘は要らないよ」）。

### 3.3 今朝の一報 (`news_headline.py`)
- ソース：NHK総合RSS (`https://www3.nhk.or.jp/rss/news/cat0.xml`) のトップ1件。
- スコアリング・ラベル分類・トレンド検出はしない（シンプルにトップ記事のタイトル＋リンク）。
- リンクは付けるが、要約はMVPでは不要（タイトルのみ）。失敗時はブロックを省略。

### 3.4 本日の一語 (`daily_word.py`)
- Cohere (`command-r-plus-08-2024`) に毎朝「今日の一語」を生成させる。
- ジャンルは **ミックス**（四字熟語・故事成語・哲学/思想用語・難読語・概念語などをランダムに）。
- 中身は「読み・意味・由来（または一言深掘り）」。
- 既出語は `data/seen_words.json` に蓄積し、プロンプトに「これらは避けて」と渡して重複回避。
- Cohere失敗 / 上限時はこのブロックをスキップ。

### 3.5 送信・整形 (`secretary.py`, `line_send.py`)
- `line_send.py`：既存 `send_line()` を流用（LINE push、4900字分割）。
- `secretary.py`：各モジュールを呼び、フランク秘書口調で1通に整形して送信。
- 1日1通。重複送信防止：`data/sent_state.json` に当日送信済みフラグ。

## 4. アーキテクチャ

| モジュール | 役割 | 依存 |
|---|---|---|
| `secretary.py` | 司令塔。各ブロック収集→整形→送信。重複送信防止。 | 下記すべて |
| `calendar_events.py` | iCal取得→該当予定抽出（繰り返し対応） | `CALENDAR_ICAL_URL` |
| `weather.py` | Open-Meteoで2地点の天気取得 | なし(無料) |
| `daily_word.py` | Cohereで一語生成・既出回避 | `COHERE_API_KEY` |
| `news_headline.py` | NHK RSSトップ1件 | なし |
| `line_send.py` | LINE push（既存ロジック流用） | `LINE_ACCESS_TOKEN`, `LINE_USER_ID` |

各モジュールは「何をするか／どう呼ぶか／何に依存するか」が明確な単一責務。
単体でテスト可能（例：`python calendar_events.py` で抽出結果をstdout表示）。

## 5. データの流れ

1. cron 5:30 → `secretary.py` 起動
2. 各モジュールが情報取得（カレンダー／天気／ニュース／一語）
3. フランク秘書口調で1通に整形
4. LINE push
5. `data/sent_state.json` に当日フラグ記録

## 6. エラー処理

- 各ブロックは独立。失敗したブロックは省略 or 「（◯◯を取得できなかったよ）」と表示し、他は継続。
- カレンダー取得失敗 → 予定ブロックにエラー注記、天気・一語等は通常通り。
- Cohere上限/失敗 → 一語ブロックをスキップ。
- 全ブロック失敗時でも挨拶だけは送る（疎通確認になる）。
- ログは既存 `logs/` ディレクトリへ。

## 7. 退役・流用

- **消す**：スコア学習(`scores.json`)、👍👎クイックリプライ、fetch/sendの2段、トレンド検出、20ラベル分類、`request_counter` のニュース用途。
- **残す/流用**：`.env`、`venv`、LINE送信ロジック、`git pull`→cron実行の仕組み、`logs/`、Cohereクライアント設定。
- 旧 `news_delivery.py` は削除せず `archive/` 等へ退避（参照用）。cronからは外す。

## 8. cron

旧3行（fetch 3時 / send 7時 / status 23:55）を削除し、新1行に置換：

```
30 5 * * * cd <SCRIPT_DIR> && git pull --quiet && <PYTHON> secretary.py >> logs/secretary.log 2>&1
```

`setup_cron.sh` を新仕様に書き換える。

## 9. 新規依存

- `icalendar`（.icsパース）
- `recurring-ical-events`（繰り返し予定の展開）

## 10. 設定 (`.env` 追記)

```
CALENDAR_ICAL_URL=<GoogleカレンダーのiCal秘密URL>
```

天気の座標・地点名はコード内定数（自宅=座間, 職場=港区）。

## 11. スコープ外（YAGNI）

- パーソナライズ学習・👍👎フィードバック（廃止）
- 複数日先の天気詳細（今日＋明日まで）
- カレンダー書き込み
- 予定の時刻ぴったり通知（朝の通達にまとめるため不要）
- ニュースの要約・複数件（トップ1件のみ）
